# Research Report: Copyright-Proofing APIs and Services for AI-Generated Content

## Overview
This report examines available APIs and services for establishing copyright proof and provenance for AI-generated content. The focus is on free/open-source solutions and affordable options that provide legal credibility and technical verification.

## Key Findings

### Legal Context (U.S. Copyright Law)
- **Purely AI-generated works** (even with detailed prompts) are **not protected by copyright** due to lack of human authorship
- **AI-assisted works** with substantial human creative contribution **can be eligible for copyright**, but only the human-authored components are protected
- **Documentation requirement**: Must retain proof of creative input (drafts, timestamps, revisions, version history)

### Technical Solutions

#### 1. Content Watermarking & Attribution

**Google DeepMind SynthID-Text**
- **Type**: Open-source watermarking tool
- **Cost**: Free
- **Features**: Embeds invisible, detectable markers in AI-generated text
- **Use Case**: Prove content was generated by your system
- **Source**: [Axios Article](https://www.axios.com/2024/10/24/google-deepmind-ai-text-watermarking-tool)

**Guardians of Generation**
- **Type**: Open-source framework
- **Cost**: Free
- **Features**: Inference-time detection and prompt rewriting to prevent copyrighted content
- **Use Case**: Prevent copyright infringement in image generation
- **Source**: [arXiv Paper](https://arxiv.org/abs/2503.16171)

**Protect-Your-IP**
- **Type**: Research framework
- **Cost**: Free (research implementation)
- **Features**: Scalable watermark and attribution for images with traceable decoding
- **Use Case**: Verify origin and attribution of generated images
- **Source**: [arXiv Paper](https://arxiv.org/abs/2405.16596)

#### 2. C2PA (Coalition for Content Provenance and Authenticity)

**C2PA Reference Implementation**
- **Type**: Open-source standard
- **Cost**: Free
- **Languages**: Rust (with WASM, Python, Node.js bindings)
- **Features**:
  - Create, read, and verify content credentials
  - Supports JPEG, PNG, MP4, etc.
  - CLI and SDKs for integration
  - Tamper-evident provenance records
- **GitHub**: [c2pa-org/c2pa-rs](https://github.com/c2pa-org/c2pa-rs)
- **Documentation**: [C2PA.org](https://c2pa.org/specifications/)

**Adobe Content Authenticity Initiative (CAI)**
- **Type**: Commercial/free verification
- **Cost**: Free for verification
- **Features**: Browser and API-based credential verification
- **Website**: [contentcredentials.org](https://contentcredentials.org/)

**Truepic Foresight**
- **Type**: Commercial API
- **Cost**: Enterprise pricing (contact sales)
- **Features**: 
  - Extracts and verifies C2PA credentials
  - REST API for enterprise use
  - Independent verification of provenance signals

**Typical C2PA API Workflow**:
```http
POST /v1/verify
Content-Type: multipart/form-data

file=@photo.jpg

Response:
{
  "authentic": true,
  "signedBy": "creator.com",
  "provenance": [
    {"action": "generated", "tool": "DALL-E", "time": "2024-11-09T17:04Z"},
    {"action": "edited", "software": "Photoshop", "time": "2024-11-09T18:20Z"}
  ],
  "tampered": false
}
```

#### 3. Blockchain Timestamping Services

**Sunlit Daymark**
- **Cost**: **Free**
- **Blockchain**: Bitcoin
- **Features**:
  - Tamper-proof timestamps
  - PDF certificates and cover sheets
  - Email notifications
  - Proof files (.ots) for independent verification
  - Privacy-first (only SHA-256 hash used)
- **Website**: [daymarks.app](https://daymarks.app/)
- **Best For**: Creators seeking cost-free, credible proof

**Digital Timestamps (DTS)**
- **Cost**: **$2.50 per file**
- **Blockchain**: Bitcoin (OpenTimestamps)
- **Features**:
  - Sub-second proof generation
  - Lifetime verification
  - No account needed
- **Website**: [digitaltimestamps.com](https://digitaltimestamps.com/)
- **Best For**: Small batch, affordable use

**Stampit**
- **Cost**: **< €1 per stamp**
- **Blockchain**: Multiple low-carbon blockchains
- **Features**:
  - Flexible API
  - Privacy-first (files stay local)
  - Smart contract verification
- **Website**: [stampit.io](https://stampit.io/)
- **Best For**: High-volume stamping, budget-conscious users

**Eternity Bits**
- **Cost**: **$9 per file** (one-time)
- **Blockchain**: Bitcoin
- **Features**:
  - Developer-friendly API
  - Permanent on-chain proof
  - Privacy-first local hashing
  - GitHub integration
- **Website**: [eternitybits.com](https://eternitybits.com/pricing/)
- **Best For**: Developers needing automation

**Evistamp**
- **Cost**: Contact sales (affordable)
- **Blockchain**: Multi-chain (VeChain, Celo, Polygon, Elrond)
- **Features**:
  - Real-time timestamping
  - Scalable API
  - Forensic integrity reports
- **Website**: [evistamp.com](https://www.evistamp.com/)
- **Best For**: Multi-chain support needs

**time:beat (by Enigio)**
- **Cost**: **Free for personal use**
- **Blockchain**: Bitcoin, Ethereum, Twitter, and more
- **Features**:
  - eIDAS-aligned
  - Digital notary functionality
  - Scalable and legally minded
- **Website**: [enigio.com/timebeat](https://enigio.com/timebeat/)
- **Best For**: Legal-aligned timestamps

**Autoproof**
- **Cost**: $49/month (startup), $199/month (mid-tier), $470/year (20% off)
- **Features**:
  - Codebase copyright protection
  - CI/CD integration
  - Blockchain-stamped certificates
- **Website**: [autoproof.dev](https://autoproof.dev/codebase)
- **Best For**: Companies managing code IP claims

**TimeProofs**
- **Cost**: Low predictable cost
- **Type**: Non-blockchain alternative
- **Features**:
  - Signed timestamps via open JSON standard (ProofSpec)
  - Instant, edge-native verification
  - Privacy-first
- **Website**: [timeproofs.io](https://timeproofs.io/)
- **Best For**: Privacy-conscious workflows without blockchain

## Recommendations

### For Individual Creators
1. **Free Option**: Use **Sunlit Daymark** for Bitcoin-based timestamping with professional certificates
2. **Watermarking**: Implement **SynthID-Text** for text content or **Protect-Your-IP** for images
3. **Provenance**: Adopt **C2PA Reference Implementation** for comprehensive content credentials

### For Small Businesses
1. **Affordable Timestamping**: **Digital Timestamps** ($2.50/file) or **Stampit** (< €1/stamp)
2. **Content Verification**: **Adobe CAI** for free verification services
3. **High Volume**: **Stampit** for scalable, low-cost stamping

### For Developers
1. **API Integration**: **Eternity Bits** ($9/file) with GitHub integration
2. **Code Protection**: **Autoproof** for CI/CD workflows
3. **Custom Implementation**: **C2PA Reference Implementation** (Rust/Python/Node.js)

### For Enterprise
1. **Commercial Solutions**: **Truepic Foresight** for enterprise-grade C2PA verification
2. **Multi-Chain**: **Evistamp** for diverse blockchain support
3. **Compliance**: **time:beat** for eIDAS-aligned timestamping

## Implementation Notes

### Best Practices
1. **Document Creative Process**:
   - Save prompt inputs, intermediate versions, edits
   - Maintain timestamps and revision history
   - Use version control with labels

2. **Implement Watermarking**:
   - Text: SynthID-Text for invisible markers
   - Images: Protect-Your-IP or Guardians of Generation
   - Apply at generation time

3. **Establish Provenance**:
   - Use C2PA for comprehensive content credentials
   - Embed metadata with creation details
   - Sign with cryptographic keys

4. **Blockchain Timestamping**:
   - Hash content locally (SHA-256)
   - Submit hash to timestamping service
   - Store proof files (.ots, certificates)
   - Never upload actual content (privacy)

5. **Maintain Audit Trail**:
   - Log all generation events
   - Track human modifications separately
   - Use HMAC for usage records
   - Keep evidence for legal protection

### Technical Integration Patterns

**Watermarking Workflow**:
```
1. Generate AI content
2. Apply watermark (SynthID-Text/Protect-Your-IP)
3. Store watermarked version
4. Maintain detection keys securely
5. Verify watermark on demand
```

**C2PA Workflow**:
```
1. Generate content
2. Create manifest with provenance data
3. Sign manifest with private key
4. Embed manifest in content file
5. Distribute content with credentials
6. Verify via C2PA API/tools
```

**Blockchain Timestamping Workflow**:
```
1. Generate content
2. Hash content locally (SHA-256)
3. Submit hash to timestamping API
4. Receive proof/certificate
5. Store proof with content metadata
6. Verify timestamp independently
```

### Security Considerations
- Never upload original content to third-party services (use hashes)
- Maintain private keys securely for signing
- Use multiple verification methods for critical content
- Regular backup of proof files and certificates
- Implement access controls for watermark detection keys

### Legal Compliance
- Clearly separate human-authored vs AI-generated components
- Maintain documentation for copyright registration
- Label AI-generated content per OpenInfra policy
- Ensure AI tool licensing compatibility
- Prepare evidence of creative control
